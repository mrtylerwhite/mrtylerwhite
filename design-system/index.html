Internal design system audit

This page documents the current HTML/CSS system extracted from a hardcoded WordPress/Divi export. It inventories foundations, layout patterns, and reusable components as they exist today, highlights inconsistencies, and recommends cleanup priorities—without introducing new frameworks (no Tailwind).

Last audited 2026-05-10 Scope HTML + inline CSS + plugin CSS in repo Note Production pages contain large Divi inline CSS not duplicated here

Foundations

Baseline tokens inferred from inline CSS present in exported pages, plus site-specific overrides (notably the Newsletter page).

Colors

What exists today
A Divi/WP baseline palette (e.g. link/accent #2ea3f2) plus page-specific accents (Newsletter hover/focus #c59060, CTA accent #24c8dc).
Where it appears
Global-ish defaults are in head inline styles across most pages (e.g. index.html, case-studies/index.html). Newsletter includes a dedicated style block (newsletter/index.html).
CSS selectors/classes
a, .et_pb_button, #main-header, #main-footer, .newsletter-input-container
Accent / Link (Divi default)
#2ea3f2
Footer background

Newsletter accent (hover/focus ring)

Known issues
  • Near-duplicate accents exist across pages (e.g. #2ea3f2 vs Newsletter accent #c59060 vs CTA accent #24c8dc).
  • Some Newsletter cards use one-off inline styles (e.g. borders/shadows in style="" attributes).

Cleanup recommendation: define a small accent set (Primary/Secondary/Neutral) and migrate one-off inline colors to named variables (even if only within Newsletter CSS initially).

Typography

What exists today
Baseline body: Open Sans, 14px, line-height ~1.7 (Divi default). Headings use fixed sizes: H1 30px, H2 26px, H3 22px, etc (Divi default). Newsletter introduces its own hero title and supporting text sizing.
Where it appears
Divi defaults in the head inline style block on most pages. Newsletter overrides in <style id="et-builder-module-design-deferred-newsletter">.
CSS selectors/classes
body, h1..h6, .newsletter-hero-title, .newsletter-hero-proof, .newsletter-hero-lede

Known issues
  • Heading sizes are fixed, and column-specific heading overrides exist (e.g. .et_pb_column_1_3 h1 etc in Divi CSS).
  • Newsletter typography can diverge noticeably from the rest of the site (intentionally), but currently mixes class-based CSS with inline styles.

Cleanup recommendation: define a typography scale (H1/H2/H3/body/caption/eyebrow/button) and normalize heading overrides to component-level classes rather than layout-based selectors.

Typography utilities (Phase 5)

Reusable, opt-in classes are defined in /assets/css/system/typography.css, which index.css imports. Stylesheets load only from the document <head>—this section contains documentation only (no <link rel="stylesheet"> here).

Utilities apply only when added to markup—no global h1–h6 or p overrides.

Purpose
Normalize hero, page, section, and card titles plus body/caption/meta/link treatments using design tokens, clamp() for key headings, and mobile-safe line lengths (no fixed widths on utilities).

When to use each utility

.ds-hero-title
Primary headline in heroes (homepage video hero, newsletter hero, large promotional bands).
.ds-page-title
Main title for interior pages (about, case studies index, resume) when migrating off Divi default heading sizes.
.ds-section-title
Major section headings inside long pages (mid-page bands, “What readers say”, feature grids).
.ds-card-title
Titles inside cards and tiles; aligns with .ds-card__title in spirit for non-card markup.
.ds-eyebrow
Kicker labels above headlines (category, series name, “New issue”).
.ds-body / .ds-body--large / .ds-body--small
Default paragraph density; large for intros and hero supporting copy; small for dense UI paragraphs.
.ds-caption
Figure captions, thumbnail labels, supplementary notes.
.ds-meta
Secondary lines: dates, bylines, roles, taxonomy—paired well with listings and card footers.
.ds-text-link
Inline text links where underline affordance should match the tokenized link color (not button-styled CTAs).

Areas to migrate later (not done in Phase 5)

  • Global Divi head styles — h1–h4 fixed pixel sizes in inline blocks on most pages → apply utilities in markup where markup is controlled.
  • Column-specific heading overrides — e.g. .et_pb_column_1_3 h1 patterns → replace with component classes (.ds-section-title, etc.) when templates are refactored.
  • Newsletter hero — .newsletter-hero-title, .newsletter-hero-lede, proof lines → candidates for .ds-hero-title, .ds-body--large, .ds-meta once explicitly migrated.
  • Case study long-form — pullquotes, stats, image captions → .ds-caption, .ds-body--small.
  • Footer / legal — copyright and secondary lines → often .ds-caption or .ds-meta.
Known typography inconsistencies (from audit)
  • Fixed heading sizes in Divi defaults conflict with layout-based column overrides (.et_pb_column_* h1).
  • Newsletter typography intentionally diverges but mixes class-based CSS with inline styles.
  • Multiple accent/link behaviors (#2ea3f2 vs newsletter focus #c59060) affect perceived “primary link” color.
  • Body text is broadly consistent (14px / 1.7) but module-level exceptions are common.
  • See also Inconsistencies (inventory) — heading sizes, duplicate components, and spacing layers compound type rhythm drift.

Spacing

What exists today
Divi sections use percentage padding on desktop and fixed pixel padding on mobile/tablet. Many page-specific modules add one-off margins/padding (e.g. 4.5em, 2em, 50px).
Where it appears
Divi base: .et_pb_section, .et_pb_row. Per-page: et-builder-module-design-deferred-* style blocks near the bottom of each HTML file.
Selectors/classes
.et_pb_section, .et_pb_row, .et_pb_module, plus per-page .et_pb_section_0, .et_pb_row_1, etc.
Cleanup recommendation
  • Extract a spacing scale from existing values (suggest: 4, 8, 12, 16, 24, 32, 48, 64) and map the most common px/em/% values onto it.
  • Reduce one-off inline spacing on Newsletter cards into a single component style.

Where it appears
Inline CSS in nearly every HTML page.

What exists today

Known issues

Component name
Newsletter testimonial card
Type
Molecule

Reusable utilities (Phase 8)
.ds-testimonial, .ds-testimonial__stars, .ds-testimonial__quote, .ds-testimonial__author, .ds-testimonial__avatar, .ds-testimonial__name, .ds-testimonial__role
CSS file
/assets/css/system/components/testimonials.css

When to use

Compose with .ds-card (typically with the .ds-card--testimonial variant) for the surface, then add .ds-testimonial on the same root element to apply masonry-safe behavior, an editorial serif quote treatment, and an initials-based avatar circle.

  • Avatar: when no image asset exists, place 1–3 initials directly inside .ds-testimonial__avatar. To use an image instead, drop an <img> as the only child of the avatar; it is sized and clipped automatically.
  • Author meta: name and role render as block elements; wrap them in any element you need to stack them next to the avatar. The newsletter uses .newsletter-t-meta as that wrapper.
  • Masonry: the root carries break-inside: avoid + display: inline-block; vertical-align: top, so cards never split inside a CSS multi-column parent. Outside column contexts (grid/flex) those properties are no-ops.

Responsive rules

Breakpoints seen
981px, 980px, 768px, 767px, 600px, 479px
Major component changes
Header switches to mobile nav below 980px; rows/columns collapse to single column; some modules hide on tablet/phone using page-specific rules.
Risk areas
  • Conflicting page-specific media queries (especially when mixing 768 and 767 cutoffs).
  • Hard-coded widths/max-widths inside page-scoped selectors (.et_pb_row_* etc).

Inconsistencies (inventory)

  • Duplicate button styles: Divi buttons vs Newsletter submit button system.
  • Duplicate card styles: Divi row-as-card vs Newsletter custom cards + inline styles.
  • Duplicate nav/header patterns: menu items styled as buttons; hover/focus behavior varies.
  • Inconsistent heading sizes: fixed heading sizes plus layout-based overrides.
  • Inconsistent form styles: Newsletter pill input + focus ring differs from Divi inputs.
  • One-off colors: multiple accents and inline swatches.
  • One-off spacing: frequent em/px/% mix and inline spacing.
  • Conflicting media queries: repeated 980/768/767/600/479 rules across different layers.
  • Legacy WordPress CSS: WP preset variables and block styles present even when not used widely.
  • Overly specific selectors: page-scoped .et_pb_row_*, .et_pb_section_* increase maintenance cost.
  • Hardcoded widths: max-width overrides (e.g. row max-width 750px on homepage) can create inconsistent rhythm.

Cleanup priorities

High
  • Mobile-breaking layout rules (audit page-scoped media queries; fix overlaps).
  • Duplicated header/nav patterns (semantics + consistent CTA treatment).
  • Form styles needed for Newsletter (standardize input + submit button).
  • Buttons and CTAs (single base + variants across site + newsletter).
  • Core typography scale (normalize headings/eyebrows/button text).
Medium
  • Card normalization (case study + newsletter cards).
  • Testimonial styles (extract into shared component).
  • Logo strips (standardize spacing/alignment).
  • Section spacing rhythm (reduce one-offs).
  • Legacy WordPress/Divi CSS cleanup plan (identify unused selectors safely).
Low
  • Minor color cleanup (near-duplicates, token naming).
  • Animation cleanup (ensure consistent motion + reduced motion support).
  • Decorative details (borders, shadow fine-tuning).
